## System Overview

Argos is a **SvelteKit SDR & Network Analysis Console** deployed natively on Raspberry Pi 5 (Kali Linux) for Army EW training at NTC/JMRC. It wraps native CLI tools (hackrf_sweep, gpsd, Kismet, grgsm_livemon) into a real-time web dashboard with WebSocket push, MapLibre GL mapping, and MIL-STD-2525C symbology.

## Module Guide

### Server Core (`src/lib/server/`)

#### `env.ts` — Environment Configuration

Zod-validated env vars loaded at module init. Process exits on parse failure. Required: `ARGOS_API_KEY` (≥32 chars). Optional: `KISMET_*`, `ANTHROPIC_API_KEY`, `CLOUDRF_API_KEY`, `OPENCELLID_API_KEY`, `GSMEVIL_DIR`, `GPSD_SOCKET_PATH`.

#### `auth/` — Authentication

Fail-closed single-operator auth. Two paths: `X-API-Key` header (programmatic) or `__argos_session` HMAC-SHA256 cookie (browser). `timingSafeEqual` via HMAC normalization. WebSocket auth at upgrade level only.

#### `api/` — Route Handler Factory

`createHandler(fn, options)` — wraps try/catch, structured logging, optional Zod body validation, JSON response. Used by 53 of 66 routes.

#### `security/` — Security Primitives

`RateLimiter` (sliding window, in-memory), `buildCorsHeaders`, `sanitizeString`, `validateInterfaceName`, `safeJsonParse`, `logAuthEvent`.

#### `middleware/` — Request Pipeline

Rate limit middleware (429 + Retry-After), security headers (CSP, X-Frame-Options, Referrer-Policy, Permissions-Policy), WebSocket upgrade handler.

#### `db/` — SQLite Persistence

`RFDatabase` facade over `better-sqlite3`. WAL mode, 64MB cache, 128MB mmap. Tables: `devices`, `signals`, `relationships`, `spatial_grid`, `signal_stats_hourly`, `tak_server_configs`. Repositories: signal, spatial (bounding-box), network (graph edges), device. Auto-cleanup: 7-day retention, hourly aggregation, 1000-row batches. Migrations tracked in `__migrations` table. `DatabaseOptimizer` for PRAGMA tuning and slow-query tracking.

#### `hardware/` — Hardware Management

`HardwareRegistry` singleton — register, query, filter by category (sdr/wifi/bluetooth/gps/cellular/serial). Event-driven status updates. `ResourceManager` singleton — mutex-based ownership per device, orphan scan on startup + every 30s. Detection pipeline: USB (`lsusb`), network (`iw dev`), serial (`/dev/ttyUSB*`).

#### `hackrf/` — Spectrum Sweep

`SweepManager extends EventEmitter` — spawns `hackrf_sweep`, parses CSV output into `SpectrumData`, frequency cycling, exponential-backoff error recovery. Split across 4 files (<300 LOC each). Acquires resource lock before starting. `testHackrfAvailability()` via `timeout 3 hackrf_info`.

#### `kismet/` — WiFi/RF Monitoring

`KismetProxy` — HTTP client to Kismet REST API. `WebSocketManager` singleton (HMR-safe via `globalThis`) — polls every 2s, deduplicates via device cache, throttles per-device updates (500ms), broadcasts to subscribed WS clients. `FusionKismetController` — Zod-validates Kismet responses with safe fallback stub.

#### `services/gps/` — GPS Position

Unix socket client to gpsd. Sends `?WATCH={"enable":true,"json":true}`, parses TPV events. Circuit breaker on satellite service (exponential backoff). 5m dead-band filter on position updates.

#### `services/gsm-evil/` — GSM Monitoring

Control service (start/stop process/container), health probes, frequency scan pipeline, AI-guided intelligent scan, grgsm_livemon process management, tshark frame capture, GSM protocol parsing (RACH, SDCCH, BCCH, SMS).

#### `tak/` — Team Awareness Kit

`TakService extends EventEmitter` — CoT over TLS via `@tak-ps/node-tak`. Exponential reconnect (1s→30s). `TakSaBroadcaster` — sends SA position every 30s (UID `ARGOS-1`, type `a-f-G-U-C`). Cert manager (P12/PEM/CRT/KEY). TAK package parser (ZIP + manifest.xml).

#### `gsm/` — L3 Decoder

Pure GSM 04.08 Layer 3 PDU decoder. No side effects, no process spawning. Safe to call anywhere.

#### `mcp/` — MCP Servers

7 servers extending `BaseMcpServer` (stdio transport): hardware-debugger, database-inspector, system-inspector, streaming-inspector, gsm-evil, test-runner, api-debugger. `DynamicMcpServer` — hardware-gated tool list. `ArgosApiClient` — HTTP client for MCP servers running outside SvelteKit process.

#### `agent/` — AI Agent

`AgentRuntime` — sends messages to Anthropic API (Claude Sonnet 4), handles `tool_use` content blocks, dispatches to registered tools (DB queries, hardware control), loops until `end_turn` or max iterations.

### Components (`src/lib/components/`)

#### Dashboard Shell

`DashboardShell` — root layout using **Svelte 5 snippet-based composition** (not slots). `IconRail` (data-position attribute for CSS-driven positioning), `TopStatusBar` (status sub-indicators). Mode switches between sidebar and full-width.

#### Dashboard Map

`DashboardMap` + `dashboard-map-logic.svelte.ts` (Svelte 5 runes module). `$derived` GeoJSON from stores via `fromStore()` bridge. Cell towers lazy-fetched after CSS paint frame. RF propagation overlay via `syncRFOverlays()`.

#### Device Panels

`DeviceTable` (fully controlled — all data as props), `DeviceSubRows`, `DeviceToolbar`, `DevicePriorityTable` (threat prioritization), `IntelCategoryView`. Sort/filter logic in pure functions.

#### GSM Evil

Full-screen scan console. `ScanResultsTable`, `ErrorDialog`. State persisted to localStorage with 2s debounced writes, version-checked load, transient key stripping.

#### Terminal

xterm.js connected to PTY WebSocket server (`/terminal-ws`). Session reattachment — server buffers output when no client attached.

#### Agent Chat

SSE streaming via `fetch('/api/agent/stream', {method: 'POST'})`. Manual EventSource-style read. Context bridge from GPS + Kismet stores.

#### UI Primitives

shadcn-svelte via bits-ui: alert-dialog, badge, button, input, select, separator, switch, table. All use `$props()` + `$bindable` pattern.

## Conventions

- **TypeScript strict mode**, no `any` (use `unknown` + type guards)
- **Svelte 5 runes** for new code (`$state`, `$derived`, `$effect`, `$props()`, `$bindable`)
- **Legacy stores bridged** via `fromStore()` in `.svelte.ts` rune modules
- **Logic extraction**: reactive state in `.svelte.ts` modules, not inside `.svelte` files
- **Snippet-based composition** over slots in Svelte 5
- **Fully controlled children**: data + callbacks as props, stores accessed by parent only
- **createHandler factory** for 53/66 API routes
- **Zod validation** at all boundaries: API bodies, store writes, Kismet responses, DB rows
- **No barrel files** — direct imports only (exception: `src/lib/components/ui/`)
- **kebab-case files**, camelCase vars, PascalCase types/components, UPPER_SNAKE constants
- **Max 300 LOC/file**, max 50 LOC/function
- **Singleton + HMR safety** via `globalThis.__argos_*` pattern
- **Result<T,E>** for error handling in critical paths
- **Prepared statement caching** in `Map<string, Statement>` for SQLite

## Gotchas

1. **Memory is scarce**: 8GB RPi shared across Vite (~750MB), ChromaDB, Kismet, MCP servers. `svelte-check` uses ~650MB — never run two instances.
2. **`wlan0` is sacred**: Built-in WiFi must NEVER be used with Kismet (monitor mode kills connectivity).
3. **APM requires LD_PRELOAD**: `libapmsafe.so` mandatory — gfortran ABI mismatch causes heap corruption without the 64MB memory pool.
4. **`npm run dev` vs keepalive**: Dev script calls `kill-dev` first. Keepalive must call `vite-oom-protect.sh` directly to avoid restart storm.
5. **earlyoom matches `comm` name**: `vite` in avoid list never matches (comm=`node`/`esbuild`). Hence `oom_score_adj=-500` workaround.
6. **WebSocket auth at upgrade only**: `WebSocketManager.addClient()` does not re-authenticate. Auth enforced in `websocket-server.ts`.
7. **Parrot OS codename `echo`**: Literal string breaks apt repos. `resolve_debian_codename()` maps to `trixie`.
8. **claude-mem worker accumulation**: Each session spawns orphan `bun worker-service.cjs --daemon` (50–450MB each). `cleanup-stale-daemons.sh` on SessionStart is essential.
9. **`.backup`/`.deleted` files**: Stale files in `tactical-map/` and `kismet/` — not imported but still in repo.
10. **HackRF process cannot be paused**: Frequency cycling requires stop→start. Stale state must be explicitly reset.
11. **GPS circuit breakers are independent**: Satellite service has its own breaker separate from position service — they can be in different states.
12. **TAK CoT throttle**: Duplicate UIDs dropped within 1s window. Pending CoT queued (up to 1s latency, no data loss).
13. **`gridProcessor.js` hex-grid is a stub**: `processSignalsToHexGrid()` returns empty `cells: []` always.
14. **install-services.sh not yet run**: Deployment service files exist as templates but are not installed on the current system. Run `sudo bash scripts/ops/install-services.sh` to install and enable.

## Navigation Guide

**To add a new API endpoint**: Create `src/routes/api/<domain>/+server.ts`, use `createHandler()` from `$lib/server/api/create-handler.ts`, add Zod schema to `$lib/schemas/`, add types to `$lib/types/`.

**To add a new component**: Create in `src/lib/components/<family>/`, extract reactive logic to `.svelte.ts` rune module, use `$props()` pattern, bridge legacy stores via `fromStore()`.

**To add a new store**: Create in `src/lib/stores/`, use Svelte 5 `$state` for new stores, add Zod validation for hardware-sourced data.

**To add a new service**: Create in `src/lib/server/services/<domain>/`, wrap CLI tools via `execFileAsync()`, add error handling with `Result<T,E>`, register in relevant API route.

**To add a new tactical module**: Create `tactical/modules/<name>.py`, subclass `TacticalModule`, implement `_add_module_args()` + `run(args)`, output JSON via `output_result()`.

**To add a new systemd service**: Create `deployment/<name>.service` with `__SETUP_USER__`/`__PROJECT_DIR__` placeholders, add to `SYSTEM_SERVICES` array in `install-services.sh`, run installer.

**To run database migrations**: Add SQL file to `src/lib/server/db/migrations/` or TypeScript migration, run `npm run db:migrate`.

**To modify auth**: Edit `src/lib/server/auth/auth-middleware.ts` (API auth) or `src/hooks.server.ts` (middleware chain). WebSocket auth in `src/lib/server/websocket-server.ts`.

**To add hardware detection**: Add detector to `src/lib/server/hardware/detection/`, register in `hardware-detector.ts`, update `HardwareRegistry` categories if needed.
