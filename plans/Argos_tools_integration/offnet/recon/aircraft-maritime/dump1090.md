# dump1090

> **RISK CLASSIFICATION**: LOW RISK
> This tool is part of a controlled military/defense training toolkit. dump1090 is a passive ADS-B receiver that decodes aircraft transponder broadcasts on 1090 MHz. Reception of ADS-B signals is legal in most jurisdictions. No transmit capability exists. This tool performs receive-only operations using an RTL-SDR dongle.

## Tool Description

dump1090 is an ADS-B (Automatic Dependent Surveillance-Broadcast) Mode S decoder specifically designed for RTL-SDR dongles. It receives 1090 MHz transponder signals broadcast by aircraft and decodes them to extract position, altitude, speed, heading, callsign, ICAO address, and squawk codes. The tool plots aircraft on an interactive map in real time and exposes decoded data via a JSON API, making it ideal for integration into larger situational awareness platforms like Argos.

The FlightAware fork (dump1090-fa) is the actively maintained version with significant improvements over the original antirez version, including better decoding algorithms, improved web interface, enhanced Beast protocol support, and better performance on ARM platforms. dump1090-fa is the recommended variant for production deployments and is a **top-10 priority tool** for Argos integration.

Key capabilities:

- Decodes ADS-B Mode S transponder messages from aircraft
- Plots aircraft positions on an interactive web map with altitude, speed, heading, and callsign
- Provides JSON API output (`/data/aircraft.json`) for direct programmatic integration
- Supports Beast binary protocol for feeding data to aggregation services
- Built-in web interface on port 8080 for standalone use
- Network output modes: raw, Beast, SBS (BaseStation) formats
- Interactive mode with real-time terminal display
- Supports gain control, frequency correction, and advanced RTL-SDR tuning

## Docker Compatibility

### Can it run in Docker?

YES

### Docker Requirements

- RTL-SDR USB dongle passed through to the container via `--device` flag
- `librtlsdr` and `libusb` libraries inside the container
- Host must have `rtl-sdr` udev rules configured so the container can access the USB device
- Port mapping for the built-in web interface (default 8080) and Beast output (default 30005)
- Blacklist `dvb_usb_rtl28xxu` kernel module on the host to prevent kernel driver from claiming the device

### Dockerfile

```dockerfile
FROM debian:bookworm-slim AS builder

RUN apt-get update && apt-get install -y --no-install-recommends \
    build-essential \
    debhelper \
    librtlsdr-dev \
    libusb-1.0-0-dev \
    pkg-config \
    libncurses5-dev \
    lighttpd \
    git \
    ca-certificates \
    && rm -rf /var/lib/apt/lists/*

RUN git clone https://github.com/flightaware/dump1090.git /build/dump1090-fa

WORKDIR /build/dump1090-fa
RUN make BLADERF=no
FROM debian:bookworm-slim

RUN apt-get update && apt-get install -y --no-install-recommends \
    librtlsdr0 \
    libusb-1.0-0 \
    libncurses6 \
    lighttpd \
    && rm -rf /var/lib/apt/lists/*

COPY --from=builder /build/dump1090-fa/dump1090 /usr/bin/dump1090-fa
COPY --from=builder /build/dump1090-fa/public_html /usr/share/dump1090-fa/html

EXPOSE 8080 30001 30002 30003 30004 30005

ENTRYPOINT ["dump1090-fa"]
CMD ["--net", "--device-index", "0", "--fix", "--lat", "0", "--lon", "0", "--max-range", "360", "--net-http-port", "8080", "--net-ri-port", "30001", "--net-ro-port", "30002", "--net-sbs-port", "30003", "--net-bi-port", "30004,30104", "--net-bo-port", "30005"]
```

### Docker Run Command

```bash
# Basic run with RTL-SDR USB passthrough
docker run -d \
  --name dump1090-fa \
  --restart unless-stopped \
  --device /dev/bus/usb:/dev/bus/usb \

  -p 30005:30005 \

  --net \

  --lon 0.0 \

  --net-http-port 8080 \
  --net-ro-port 30002 \
  --net-sbs-port 30003 \
  --net-bo-port 30005 \

  --name dump1090-fa \

  --name dump1090-fa \
  --restart unless-stopped \

  -p 8080:8080 \
  -p 30002:30002 \
  -p 30003:30003 \
  -p 30005:30005 \
  -v dump1090-data:/run/dump1090-fa \

  --net \
  --device-index 0 \
  --fix \
  --lat 38.8977 \
  --lon -77.0365 \
  --max-range 360 \
  --gain -10
```

## Install Instructions (Native)

```bash
# ============================================
# dump1090-fa Native Install on Kali Linux RPi5
# ============================================

# Install build dependencies
sudo apt-get update
sudo apt-get install -y \
  build-essential \
  debhelper \
  librtlsdr-dev \
  libusb-1.0-0-dev \
  pkg-config \
  libncurses5-dev \
  lighttpd \
  git

# Blacklist the kernel DVB driver so RTL-SDR is available
echo "blacklist dvb_usb_rtl28xxu" | sudo tee /etc/modprobe.d/blacklist-rtlsdr.conf
sudo modprobe -r dvb_usb_rtl28xxu 2>/dev/null || true

# Clone the FlightAware fork (recommended over original)
cd /opt
sudo git clone https://github.com/flightaware/dump1090.git
cd dump1090

# Build for ARM64 (BladeRF and PlutoSDR not needed)
sudo make BLADERF=no
# Install the binary (make produces 'dump1090'; rename to 'dump1090-fa' by convention)
sudo cp dump1090 /usr/local/bin/dump1090-fa
sudo mkdir -p /usr/share/dump1090-fa/html
sudo cp -r public_html/* /usr/share/dump1090-fa/html/

# Create run directory
sudo mkdir -p /run/dump1090-fa

# Test the installation
dump1090-fa --help

# Run in network mode with web interface
dump1090-fa --net --device-index 0 --fix --gain -10 \
  --net-http-port 8080 --net-ro-port 30002 --net-sbs-port 30003 \
  --net-bo-port 30005 --max-range 360

# Verify RTL-SDR is detected
rtl_test -t

# Access the web interface at http://<pi-ip>:8080
# Access JSON API at http://<pi-ip>:8080/data/aircraft.json
```

### RPi5-Specific Notes

- dump1090 is the quintessential Raspberry Pi application - the combination of Pi + RTL-SDR + dump1090 is the most common ADS-B receiver setup worldwide
- The RPi5 Cortex-A76 cores are vastly overpowered for dump1090 workloads; even the Pi Zero handles it well
- USB 3.0 on RPi5 provides better throughput for the RTL-SDR dongle than USB 2.0 on older Pis
- With the RPi5's 8GB RAM, dump1090 memory usage is negligible
- Can run alongside tar1090 and readsb on the same Pi without issues

### Argos Integration Notes

- JSON API endpoint `/data/aircraft.json` provides structured aircraft data updated every second
- Beast binary output on port 30005 can feed data to other aggregation tools
- SBS/BaseStation output on port 30003 provides CSV-formatted position reports
- Web interface can be embedded in Argos dashboard via iframe or data can be consumed via the JSON API
- Aircraft data includes: ICAO hex address, callsign, altitude, ground speed, heading, latitude, longitude, vertical rate, squawk, track, messages count, seen timestamp, RSSI
