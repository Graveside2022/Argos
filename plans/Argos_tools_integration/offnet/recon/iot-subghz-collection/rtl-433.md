## Tool Description

RTL-433 is a generic data receiver for ISM band devices operating on 433.92 MHz, 315 MHz, 868 MHz, and 915 MHz frequencies. It supports over 280 device protocol decoders covering weather stations, tire pressure monitors (TPMS), security alarm sensors, energy meters, medical devices, and other wireless IoT equipment. Output formats include JSON, CSV, UDP, MQTT, and InfluxDB, making it highly suitable for integration into data pipelines and dashboards.

## Docker Compatibility Analysis

### Can it run in Docker?

**YES** - RTL-433 runs cleanly in Docker. It is a lightweight CLI application with minimal dependencies. Only requires USB passthrough for the RTL-SDR dongle.

### Host OS-Level Requirements

- `--device=/dev/bus/usb` - USB passthrough for RTL-SDR dongle
- `--privileged` - Required for raw USB device access to the RTL-SDR hardware
- No special kernel modules required beyond standard USB drivers (`dvb_usb_rtl28xxu` should be blacklisted on host to prevent kernel driver conflicts)
- Host `udev` rules for RTL-SDR device permissions (typically auto-configured by `rtl-sdr` package)

### Docker-to-Host Communication

- MQTT output: Container can publish decoded signals directly to an MQTT broker on the host or network via `-F mqtt://host.docker.internal:1883`
- JSON output via volume mount: `-v /host/output:/output` for file-based logging
- UDP output: Can stream JSON to a host UDP listener via `-F json://host-ip:port`
- HTTP output: Can push to InfluxDB or other HTTP endpoints for time-series storage
- No inbound port mappings required (RTL-433 is a producer, not a server)

## Install Instructions (Docker on Kali RPi 5)

### Option A: Native Install (Recommended - available in Kali repos)

```bash
sudo apt update
sudo apt install -y rtl-433

# Blacklist kernel DVB driver to prevent conflicts
echo 'blacklist dvb_usb_rtl28xxu' | sudo tee /etc/modprobe.d/blacklist-rtlsdr.conf
sudo modprobe -r dvb_usb_rtl28xxu
```

### Option B: Docker

```dockerfile
FROM kalilinux/kali-rolling:latest

RUN apt-get update && apt-get install -y \
    rtl-433 \
    rtl-sdr \
    librtlsdr-dev \
    libusb-1.0-0-dev \
    && rm -rf /var/lib/apt/lists/*

WORKDIR /output

ENTRYPOINT ["rtl_433"]
```

```bash
# Build
docker build -t argos/rtl-433 .

# Run - listen for all ISM band devices with JSON output
docker run --rm --privileged \
  --device=/dev/bus/usb \
  -v $(pwd)/output:/output \
  argos/rtl-433 -F json:/output/signals.json

# Run - listen on 433 MHz and output to MQTT broker on host
docker run --rm --privileged \
  --device=/dev/bus/usb \
  --net=host \
  argos/rtl-433 -F mqtt://localhost:1883,retain=0,devices=rtl433/devices

# Run - listen on 915 MHz (US ISM band) with verbose output
docker run --rm --privileged \
  --device=/dev/bus/usb \
  -v $(pwd)/output:/output \
  argos/rtl-433 -f 915M -v -F json:/output/915mhz.json

# Run - specific protocol only (e.g., Acurite weather stations)
docker run --rm --privileged \
  --device=/dev/bus/usb \
  argos/rtl-433 -R 40
```

## Kali Linux Raspberry Pi 5 Compatibility

### Architecture Support

**ARM64 NATIVE** - RTL-433 is available as a pre-built package in the Kali Linux ARM64 repositories. Installs directly via `apt install rtl-433` with no compilation required. All 280+ protocol decoders are included.

### Hardware Constraints

- CPU: Extremely lightweight - uses minimal CPU for signal decoding. A single Cortex-A76 core handles all decoders simultaneously without strain
- RAM: Minimal memory usage (~20-50MB), negligible impact on the 8GB available
- SDR: Requires RTL-SDR dongle (installed on Argos). Also supports SoapySDR-compatible devices
- Storage: Log files can accumulate over time if JSON output is directed to disk; plan for periodic rotation

### Verdict

**COMPATIBLE** - RTL-433 is one of the most deployment-friendly tools in the Argos toolkit. Native ARM64 package available in Kali repos. Negligible resource consumption. Docker adds unnecessary overhead for this tool; native install is recommended. MQTT and JSON output integrate directly with Argos data pipelines.
